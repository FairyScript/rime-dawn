# AGENTS.md — 万象拼音精简版 (rime-wanxiang-slim)

## What This Is

Rime input method user data directory for the **万象拼音精简版** (Wanxiang Pinyin Slim) schema.
Source repo: `https://github.com/Fidelxyz/rime-wanxiang-slim`
Engine: Weasel (小狼毫) 0.17.4 / Rime 1.13.1 — Windows `%APPDATA%\Rime`.

This is NOT a typical software project. There are no package managers, build scripts, or test runners locally. The "build" is Rime's own `rime_deployer` (triggered by Weasel's "重新部署").

## Critical Rules

- **NEVER edit `*.schema.yaml` or `default.yaml` directly.** All user customizations go in `*.custom.yaml` files using Rime's `patch:` mechanism. The `custom/` subdirectory holds template copies.
- **NEVER edit files in `build/`.** Those are compiled binary artifacts (`.prism.bin`, `.table.bin`, `.reverse.bin`) generated by Rime's deployer. They will be overwritten on next deploy.
- **Version strings** in `wanxiang.schema.yaml` and `lua/wanxiang/wanxiang.lua` are managed by `release-please` (markers: `# x-release-please-version`). Do not manually bump.
- **Lua type annotations are mandatory.** All Lua code uses `---@param`, `---@return`, `---@class`, `---@field` EmmyLua/LuaLS annotations. Follow existing patterns.

## How to Modify Behavior

1. **Schema settings** (page size, switches, toggles): Edit `wanxiang.custom.yaml` using `patch:` syntax.
2. **Pinyin scheme**: The `speller/algebra` patch in `wanxiang.custom.yaml` references `wanxiang_algebra:/base/<scheme>`. Valid schemes: 全拼, 自然码, 智能ABC, 小鹤双拼, 微软双拼, 搜狗双拼, 紫光双拼, 国标双拼, 拼音加加, 乱序17, 自然龙, 汉心龙.
3. **Lua modules**: Each feature is a standalone file in `lua/wanxiang/`. Modules are referenced in `wanxiang.schema.yaml` via `lua_processor@*wanxiang.<name>`, `lua_filter@*wanxiang.<name>`, or `lua_translator@*wanxiang.<name>`.
4. **Dictionaries**: Add dict files to `dicts/`, then import them in `wanxiang.dict.yaml`.
5. **Custom phrases**: Create `custom_phrase.txt` in this directory (not checked in).

## Lua Module Conventions

- Each module returns a table `M` (or multiple tables for processor+filter combos like `english.lua`).
- Modules use `require("wanxiang.wanxiang")` for shared utilities and `M.RIME_PROCESS_RESULTS` constants.
- Processor functions: `M.init(env)`, `M.func(key, env)` returning `kAccepted`/`kNoop`/`kRejected`.
- Filter functions: `M.func(input, env)` iterating candidates with `for cand in input:iter()`.
- Translator functions: `M.func(input, seg, env)` yielding candidates via `yield(cand)`.
- Configuration is read from `env.engine.schema.config` via `wanxiang.get_config()`.

## Pinyin Algebra

`wanxiang_algebra.yaml` contains ~2800 lines of transformation rules organized by:
- `base:` — per-scheme rules (full pinyin, double-pinyin variants, fuzzy initials/finals)
- `模糊音:` — optional fuzzy sound rules (commented out by default)

Rules use Rime's `xform`, `derive`, `abbrev`, `erase`, `xlit` operators. The tone system uses circled numbers (①②③④) mapped to digits (7890) for tone-based filtering.

## Rebuild After Changes

After editing any `.yaml` or `.lua` file, trigger a Rime deploy:
- **Weasel**: Right-click tray icon → "重新部署" (Redeploy)
- Or wait for auto-deploy on file change (if enabled)
